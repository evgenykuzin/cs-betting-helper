## 🔑 Ключевые технические решения

1. **Модульная архитектура**
   - Легко добавлять новые источники данных (API/парсеры)
   - Унифицированная модель `Match` — независимо от провайдера

2. **TimescaleDB для временных рядов**
   - Оптимизировано для частых вставок
   - Автоматическое сжатие старых данных
   - Быстрые запросы по времени

3. **Асинхронный код (asyncio)**
   - Параллельные запросы к нескольким букмекерам
   - Не блокирует основной поток при парсинге

4. **Strategy Pattern**
   - `BaseProvider` → все провайдеры имеют одинаковый интерфейс
   - Можно подменять/добавлять провайдеры без изменения бизнес-логики

### Для продолжения:
1. Запустить БД: `docker-compose up -d`
2. Применить схему: `docker exec -i cs-betting-timescaledb psql -U cs_user -d cs_betting < db/schema.sql`
3. Реализовать `src/storage/db.py` для сохранения снимков
4. Запустить cron для периодического polling (каждые 5 минут)
